# Working rules for this project (Soma)

- Verify before reporting. Before stating any finding as fact (a bug, a root cause, "this is working as intended"), trace it through the actual code and, where possible, the actual data — don't reason from a plausible-sounding theory or a screenshot alone. If something can't be confirmed, say so explicitly ("I think" / "I haven't confirmed") instead of asserting it.
- Re-check claims that turn out wrong. If a prior explanation is contradicted by new evidence, don't defend it — re-investigate from scratch and say plainly that the earlier explanation was wrong.
- No technical jargon — plain language explanations.
- Autonomous git workflow: commit, push, and merge `main` ↔ `claude/fitbit-app-ideas-zezgxj` without asking permission each time. Never push to any other branch without asking. Never create a pull request unless explicitly asked.
